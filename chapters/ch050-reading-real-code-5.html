The unaided read

annotated_agent.py — the whole thing
# annotated_agent.py
# A complete hand-rolled agent: two tools, history trimming,
# a turn cap, and error handling around every tool call.

MAX_TURNS = 8    # hard cap on loop iterations
KEEP_LAST = 4    # history budget -- tiny, so you can watch trimming fire

# ---- 1. Tools: plain functions the loop agrees to run --------
def calculate(expression):
    """Evaluate simple arithmetic. Anything else is refused."""
    if not set(expression) <= set("0123456789+-*/(). "):
        raise ValueError("only digits and + - * / ( ) allowed")
    return str(eval(expression))

NOTES = {
    "meridian": "Invoice #482 | Meridian Traders | base amount 42000",
    "sunrise":  "Invoice #479 | Sunrise Textiles | base amount 15500",
}

def search_notes(query):
    """Return every note that mentions the query keyword."""
    hits = [note for key, note in NOTES.items() if key in query.lower()]
    if not hits:
        raise LookupError(f"no note matches {query!r}")
    return "; ".join(hits)

TOOLS = {"calculate": calculate, "search_notes": search_notes}

# ---- 2. The 'model': a scripted stand-in for a real LLM ------
SCRIPT = [
    "THOUGHT: I need the invoice amount before I can compute GST.\n"
    "ACTION: search_notes(Meridian)",

    "THOUGHT: Base amount is 42000, so 18% GST is 42000 * 0.18.\n"
    "ACTION: calculate(42,000 * 0.18)",   # a deliberate slip: the comma

    "THOUGHT: The comma broke the tool. Retry without it.\n"
    "ACTION: calculate(42000 * 0.18)",

    "THOUGHT: 7560.0. I have everything I need.\n"
    "FINAL ANSWER: GST at 18% on invoice #482 (Meridian Traders) "
    "comes to 7,560 rupees.",
]
STATE = {"calls": 0}   # a real model needs no counter; a script does

def fake_llm(history):
    """Stands in for a real API call: transcript in, one reply out."""
    reply = SCRIPT[min(STATE["calls"], len(SCRIPT) - 1)]
    STATE["calls"] += 1
    return reply

# ---- 3. Guard rails ------------------------------------------
def trim_history(history, keep_last=KEEP_LAST):
    """Cap the transcript: keep the task plus the newest messages."""
    if len(history) <= keep_last + 1:
        return history
    dropped = len(history) - (keep_last + 1)
    print(f"[trim] dropped {dropped} old message(s) to stay in budget")
    return [history[0]] + history[-keep_last:]

def run_tool(call):
    """Parse 'name(arg)' and run it. Errors become text, not crashes."""
    try:
        name = call.split("(")[0].strip()
        arg = call[call.index("(") + 1 : call.rindex(")")].strip()
        if name not in TOOLS:
            return f"ERROR: unknown tool {name!r}; try one of {sorted(TOOLS)}"
        return str(TOOLS[name](arg))
    except Exception as exc:
        return f"ERROR: {type(exc).__name__}: {exc}"

# ---- 4. The loop: the agent itself ---------------------------
def run_agent(task, max_turns=MAX_TURNS):
    history = [f"TASK: {task}"]
    for turn in range(1, max_turns + 1):
        history = trim_history(history)
        reply = fake_llm(history)
        history.append(reply)
        print(f"--- turn {turn} ---")
        print(reply)
        if "FINAL ANSWER:" in reply:
            return reply.split("FINAL ANSWER:")[1].strip()
        if "ACTION:" in reply:
            call = reply.split("ACTION:")[1].strip()
            observation = f"OBSERVATION: {run_tool(call)}"
            history.append(observation)
            print(observation)
    return "Stopped: hit max turns without a final answer."

answer = run_agent("What is 18% GST on the Meridian Traders invoice?")
print()
print("AGENT RETURNED:", answer)

Block by block: the annotated read

Now the guided pass. Four walkthroughs, one per block. Click the highlighted lines; each note tells you what the line does and, more importantly, why a production file bothers. Every note should feel boring by the time you finish it. Boring means fluent.

Block 1 — the tools and their registry

annotated_agent.py — lines 8 to 27
def calculate(expression):
A tool is an ordinary function (ch7). Nothing here marks it as special — it becomes a "tool" only by being listed in the registry below. Tool-ness is membership, not magic.
if not set(expression) <= set("0123456789+-*/(). "):
A whitelist check using set comparison (ch4): "is every character of the input inside the allowed set?" Whitelists beat blacklists — you cannot enumerate all the bad inputs a model might send, but you can enumerate the good ones.
raise ValueError("only digits and + - * / ( ) allowed")
The tool raises rather than returning a sad string (ch9). Division of labour: the tool signals failure clearly; deciding what to do about it belongs to the loop's guard rails.
return str(eval(expression))
Safe only because of the whitelist above; a production calculator would use a real expression parser. But notice the shape: string in, string out. Tools always end in text, because text is the only thing a model can read back.
NOTES = {
A dict standing in for a real data source — a database, a CRM, a folder of files. Most tools in real agents are exactly this: a thin wrapper around something that already existed.
hits = [note for key, note in NOTES.items() if key in query.lower()]
A list comprehension (ch8): scan every note, keep the ones whose key appears in the lowercased query. When a comprehension resists you, unroll it into a for-loop with an if and an append.
raise LookupError(f"no note matches {query!r}")
"Nothing found" is information the model needs, so it is raised loudly rather than returned as an empty string the model might misread as success. The !r puts quotes around the query so the model can spot stray spaces in what it sent.
TOOLS = {"calculate": calculate, "search_notes": search_notes}
The registry: a dict from names to function objects (ch4 — and functions are values you can store, ch7). Auditing an unfamiliar agent? Read this line first: it is everything the model can possibly touch.

Block 2 — the scripted model

The one honest lie in the file. A real deployment replaces this block with a network call; everything else stays.

annotated_agent.py — lines 29 to 50
SCRIPT = [
Four canned replies, one per turn. A real model would generate these; the script replays them. Everything downstream — parsing, execution, trimming — cannot tell the difference, which is why the simulation is fair.
"ACTION: calculate(42,000 * 0.18)", # a deliberate slip: the comma
The planted mistake. Models genuinely do this — format numbers with thousands separators, add units, wrap arguments in quotes. The comma will fail calculate's whitelist, and we get to watch the recovery machinery earn its keep.
STATE = {"calls": 0} # a real model needs no counter; a script does
An honesty note in code form. A real model is stateless: it infers where it is purely by reading the transcript. Our canned script cannot read, so it cheats with a counter. The transcript is the real model's only counter.
def fake_llm(history):
Same signature as production: transcript in, one reply out. To go real, this body becomes a single API call — for example Anthropic's client.messages.create(model=..., max_tokens=..., messages=history) — and nothing else in the file changes. That one-seam property is the mark of a well-shaped agent.
reply = SCRIPT[min(STATE["calls"], len(SCRIPT) - 1)]
The min() pins the index to the last scripted reply if the loop somehow calls more times than the script covers — a cheap defensive habit that keeps an IndexError from ruining the demo. Typical of code written by someone who has been paged at 2 a.m.

Block 3 — the guard rails

This is what separates a demo from something you would let near a customer. Two helpers, and they carry most of the file's production weight.

annotated_agent.py — lines 52 to 70
def trim_history(history, keep_last=KEEP_LAST):
The memory policy from ch47, reduced to seven lines: when the transcript outgrows the budget, keep the task plus the newest messages and drop the middle. Real systems summarise or embed what they drop — but every one of them is a variation on this function.
if len(history) <= keep_last + 1:
The + 1 accounts for the pinned task message at index 0. Off-by-one details like this are exactly what to check when reviewing someone else's trimmer.
return [history[0]] + history[-keep_last:]
Element 0 — the task — is always kept; the negative slice (ch4) keeps the newest tail. An agent that trims away its own mission will happily keep working on whatever the surviving messages suggest — "drifted off-task after a long run" bugs are often a trimmer missing this line.
def run_tool(call):
The buffer zone between the model's wishes and your Python. Everything the model sends passes through here, and nothing that happens in here is allowed to kill the run.
try:
Everything inside is fallible: we are parsing free text written by a language model, then executing a function on its say-so. Wrapping it all is not paranoia; it is the job description.
arg = call[call.index("(") + 1 : call.rindex(")")].strip()
Slice between the first ( and the last ) — string surgery from ch3. If the model forgets a bracket, index raises and the except below tells it so. Native tool calling (ch35) removes this parsing entirely; the safety logic stays.
if name not in TOOLS:
Models hallucinate tool names — ask any operator. Instead of a KeyError, the reply lists the real options so the model can self-correct next turn. Error messages in agents are not for humans; they are prompts.
except Exception as exc:
A broad except is usually a smell (ch9 warned you about silencing errors) — but here the error is not silenced. It is converted, loudly and with its type name, into the observation channel. Whatever went wrong, the loop lives and the model is told.
return f"ERROR: {type(exc).__name__}: {exc}"
The most production-shaped line in the file: a crash becomes data. Watch what it buys in the run — the comma fails, the model reads the error, fixes itself, and finishes. No human intervened. That self-repair is most of what people mean when they call an agent "robust".

Block 4 — the loop

You built this in chapter 46, so the walkthrough only stops at what is new: where the helpers plug in.

annotated_agent.py — lines 72 to 88
def run_agent(task, max_turns=MAX_TURNS):
The turn cap rides in as a default argument, sourced from the config constant at the top of the file. Configuration at the top, behaviour below — a file layout convention you will see everywhere.
history = trim_history(history)
The new line, and its placement is the lesson: trim immediately before the model call, because that is the moment the transcript is about to be sent — and, with a real API, paid for by the token (ch41). Trimming anywhere else protects nothing.
reply = fake_llm(history)
Still the only line in the whole file that talks to a model. When someone says "the agent made 6 LLM calls", they mean this line ran 6 times.
if "FINAL ANSWER:" in reply:
The stop condition, unchanged from chapter 46: the model decides when it is done, the loop enforces it. With native tool calling, "done" is a reply with no tool calls — different marker, same idea.
observation = f"OBSERVATION: {run_tool(call)}"
Compare chapter 46, where the loop called TOOLS[name](arg) naked. Now every execution goes through the guard rail, which guarantees a string comes back — success or ERROR — so the loop itself needs no try/except at all. Pushing danger into one well-tested helper is how real codebases stay readable.
return "Stopped: hit max turns without a final answer."
The honest exhaustion path. If the model never finishes, the agent says so instead of hanging or pretending. In production this line usually comes with a metric increment and an alert — the "agent gave up" dashboard exists in every serious shop.

Tip

Now earn the lap by breaking it. (1) In SCRIPT, change search_notes(Meridian) to lookup_notes(Meridian): the unknown-tool message coaches the model — but the canned script cannot adapt, which is exactly the gap between it and a real one. (2) Set KEEP_LAST = 2 and watch the trimmer fire earlier and harder. (3) Delete the try:/except pair in run_tool (fix the indentation) and re-run: the comma turn now kills the whole run with a traceback — the crash every production agent you operate has quietly been saving you from.

Check yourself

1. On turn 2, the model requests calculate(42,000 * 0.18) and the comma makes the tool raise a ValueError. What does this agent do?

run_tool catches the exception and returns "ERROR: ValueError: ..." as a string, which enters the history as an observation. The model reads it, drops the comma, and succeeds — self-repair without a human.

2. Why does trim_history always keep history[0], no matter how aggressive the trim?

Index 0 is the TASK message. A trimmer that drops it leaves the model with only recent chatter and no mission — how "the agent drifted off-task" bugs are born. Real systems pin the system prompt and task for the same reason.

3. To turn this demo into a real agent calling a live LLM, what has to change?

fake_llm has the same contract as production — transcript in, reply out — so swapping its body for client.messages.create(...) (or any provider's equivalent) upgrades the agent without touching tools, guard rails, or loop.

4. The model hallucinates a tool called fetch_invoice. What does this design send back?

The if name not in TOOLS branch returns "ERROR: unknown tool 'fetch_invoice'; try one of [...]". In an agent, error messages are prompts: the more usable the error, the more likely the model picks a valid tool next turn.

5. Why does the loop call trim_history immediately before the model call, rather than after it?

The transcript costs tokens at exactly one moment: when it is sent to the model. Trimming right before that moment enforces the budget on every single call.

Key takeaways

  • You just read a complete, production-shaped agent unaided — and nothing in it was new. Tools are functions in a dict, the model is one call, memory is a list with a policy, and the loop conducts. That is the Act V victory lap.
  • Error handling around tool calls is the difference between a demo and a system: run_tool converts crashes into ERROR observations, and the model repairs itself by reading them.
  • History trimming lives immediately before the model call, and the task message is pinned at index 0 — an agent that trims its own mission drifts off-task.
  • The loop is the smallest block in the file; the guard rails outweigh it. Frameworks earn their keep by shipping the tall bars, not the famous short one.
  • A well-shaped agent has exactly one seam to reality: swap fake_llm's body for a real API call and nothing else changes.
  • Carry the five landmarks — registry, one model call, buffer zone, memory policy, exhaustion path — into every agent file you open from now on, starting with the LangChain ecosystem next chapter.
